# Ubuntu服务器全自动运维监控工作流

这是一个专用于 Ubuntu 服务器的自动化运维工作流。它配置了 `// turbo-all` 标记，使得所有系统命令均可以在 AI 代理确认无破坏性后自动执行（如果您设置了授权），从而达到“实时监控日常情况并尝试解决报错”的全自动化目的。

// turbo-all

## 第一阶段：系统健康状况扫描

1. **检查 CPU 和内存占用情况**
   获取当前的系统资源占用，找出是否有进程导致资源耗尽。
   ```bash
   top -b -n 1 | head -n 15
   free -h
   ```

2. **检查磁盘空间**
   磁盘占满是服务器常见的故障原因，检查各个分区的剩余空间。
   ```bash
   df -h
   ```

3. **检查核心服务和守护进程状态**
   列出所有启动失败或者崩溃的系统服务。
   ```bash
   systemctl --failed
   ```

4. **扫描系统错误日志**
   从系统日志和内核日志中提取最近的严重错误（Error / Critical）。
   ```bash
   journalctl -p 3 -xb -n 50
   ```

## 第二阶段：智能诊断与自动修复尝试

5. **诊断上述收集到的异常信息**
   AI 代理会分析第一阶段中获取的资源占用和错误日志：
   - 如果发现有崩溃的服务，记录下服务名称。
   - 如果发现磁盘空间不足（如使用率 > 90%），准备执行清理方案。
   - 如果发现 OOM (Out Of Memory) 杀死了进程，准备分析内存泄漏的情况。

6. **尝试重启失败的服务 (针对 systemctl --failed 发现的服务)**
   如果有服务处于 failed 状态，尝试将其重启，并随后检查其运行日志验证是否恢复。
   ```bash
   # 若有故障服务，请替换 <service-name> 为实际服务名并执行
   # sudo systemctl restart <service-name>
   # sudo systemctl status <service-name>
   ```

7. **自动清理磁盘空间 (当发现磁盘空间不足时)**
   执行一些安全的 Ubuntu 系统包清理和旧日志清理，以快速释放磁盘空间。
   ```bash
   sudo apt-get autoremove -y
   sudo apt-get clean
   sudo journalctl --vacuum-time=3d
   ```

## 第三阶段：验证与结果报告

8. **回归验证**
   执行修复动作后，重新检查报错日志或失败的服务，确认之前的异常是否已经消失。
   ```bash
   systemctl --failed
   ```

9. **生成最终运维报告**
   总结当前服务器的健康状态、发现的问题以及自动修复的结果，汇报给最终用户。
